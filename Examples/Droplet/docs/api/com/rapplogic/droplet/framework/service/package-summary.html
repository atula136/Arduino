Package com.rapplogic.droplet.framework.service

Interface Summary
OneTimeService Describes a push service that runs once, after the initial Delay, then never again
PullService Describes a user-initiated service.
RecurringService Describes a push service that runs periodically.
 

Class Summary
Alert Describes a Message sent by a Push Service to a remote XBee.
DelayedAlert Describes an alert that is schedule to be sent in the future.
DelayedAlertPushService Describes push service that schedules one or more DelayedAlert to be sent to the remote at some time in the future.
PushService Describes a service that runs once or on a periodic basic and sends content to the LCD for display.
RealtimeAlertPushService Describes a push service that sends an Alert message immediately to the remote LCD
RunnablePushService Describes a push service that requires a dedicated thread to run all the time You send alerts to a remote by calling getServiceContext().sendAlert(Alert)
SimpleRealtimeRecurringPushService Skeleton class for a Recurring Realtime Push Service.
 

Enum Summary
RecurringService.RecurringType FIXED_DELAY services will drift over time by the cumulative execution time + delay FIXED_RATE won't drift but will make up missed calls due to thread delay and/or computer sleep/hiberate
